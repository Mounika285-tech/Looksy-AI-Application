# 1. Give User Flows (Very Important)

AI agents often know the screens but don't know how they connect.

Example:

```text
Splash
→ Welcome
→ Login / Signup
→ Personalization Setup
→ Home

Home
→ Upload Items
→ AI Detection
→ Save Item
→ Home

Home
→ Outfits
→ Mix & Match
→ Outfit Result

Home
→ Outfits
→ Weather Styling
→ Outfit Result

Home
→ Planner
→ Custom Planner
→ Calendar

Home
→ Favorites
→ Outfit Details
```

This prevents the agent from creating disconnected screens.

# 3. Define AI Logic

Don't just say "AI Recommendation."

Tell the agent exactly what should happen.

### Mix & Match

```text
User selects:
- Blue Shirt

AI finds:
- Matching Pants
- Matching Shoes
- Matching Watch

Return full outfit.
```

### Weather Styling

```text
Fetch weather.

If rainy:
recommend jackets, boots

If hot:
recommend light clothes

Use wardrobe items only.
```

### Color Matching

```text
User selects item.

AI suggests colors based on
fashion color harmony rules.
```

# 4. Define MVP vs Future Features

Tell Anti Gravity:

### Build Now

* Authentication
* Wardrobe
* AI Detection
* Outfits
* Planner
* Favorites

### Future Versions

* Shopping Integration
* Social Sharing Community
* Packing Assistant
* Laundry Tracking
* Analytics

Otherwise the agent may start building unnecessary features.

# 6. Define Outfit Rules

Example:

```text
Outfit should contain:

1 Top Wear
1 Bottom Wear
1 Footwear

Optional:

Bag
Accessory
```

This helps the AI generate logical outfits.

# 9. Create a Product Rules Document

This is the one thing that prevents most mistakes.

Example:

```text
Rule 1:
AI should recommend only items that exist in the user's wardrobe.

Rule 2:
User must review AI detection before saving.

Rule 3:
Weather styling automatically fetches weather.

Rule 4:
All outfit result screens must have:
- Favorite
- Add to Planner
- Regenerate

Rule 5:
Planner reminders are optional.

Rule 6:
Bottom navigation always contains:
Home | Upload Items | Outfits | Planner | Favorites
```
